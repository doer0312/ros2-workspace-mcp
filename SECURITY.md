# Security Policy

## Supported versions

The `0.1.x` release line currently receives security fixes.

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting flow:

https://github.com/doer0312/ros2-workspace-mcp/security/advisories/new

Do not disclose exploit details in a public Issue before a fix is available. If private reporting
is unavailable, contact the repository owner through a private channel shown on their GitHub
profile.

Path sandbox escapes, unsafe symbolic-link handling, unexpected workspace code execution, raw-file
exposure, stdout protocol injection, and denial-of-service inputs are in scope. Include a minimal
reproduction, affected version, platform, and impact without attaching sensitive user workspaces.
